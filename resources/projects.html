Октябрь 2023 - наше время

Разработка Telegram-бота. Телеграмм-бот - это один из фронт-энд-сервисов для онлайн деятельности суши ресторана.
Бот интегрируется через API с backend, получая и отправляя данные для предоставления функциональных требований. Пользователь может сделать, оплатить и отслеживать свой заказ с помощью бота, а также связываться с администраторами.
Описание

Функционал бота делится на 2 роли пользователй и 3 уровня программного кода:
Роли пользователей:

  • Клиент - может зарегистрироваться/авторизоваться, а дальше заказывать, платить, отслеживать заказ. В случай неисправностей или ошибок, или иных обстоятельств - пользователю предоставляется возможность связаться с администратором и обсудить проблему;
  • Администратор - модерирует поступаемые заявки от клиентов: принимает или отклоняет. Может обновлять распорядок, отслеживать пользовательские запросы, пожелания, жалобы;

Программные уровни:

  • API - программный интерфейс для взаимодействия с backend сервисом для выполнения операций, запрашиваемых клиентом на уровне фронт-энда;
  • Service - уровень приложения, который сериализует получаемые и отправляемые данные в вид, понятный боту и backend соответственно;
  • Bot - уровень приложения, в котором реализована логика взаимодействия с пользователем за счет возможностей, предоставленных фреймворком Aiogram 3.x;

В ходе разработки были использованы: Docker, Python (Aiogram 3.x, requests...) - как технологии для разработки и развертывания.
А так же принципы чистого кода, SOLID, паттерны проектирования: абстрактная фабрика.

Октябрь 2023 - октябрь 2023

Разработка и поддержка. Разработал Telegram-бота, который по входным данных составлял договор-заявку для заказчика и перевозчика. Проект был реализован для ИП, занимающееся логистикой грузоперевозок.
Описание

В разработке использовал язык программирования Python. Мне необходимо было построить машину состояний, которая будет запрашивать информацию в виде 30 вопросов. После получения информации, бот производил заполнение документов по указанным шаблонам Jinja и библиотеки работы с файлами типа .docx. После чего пользователю высылается 2 заполненных документа-заявки.

Функционал так же предусматривает сохранения заявок на сервере, возможность получения любой созданной заявки, удаление одной заявки, удаление всех заявок и кнопку "помощь". Операции выполняются в соответствии с названием.
